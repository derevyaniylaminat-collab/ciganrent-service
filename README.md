## 🏗️ Архітектура проєкту (виконано Студентом 1)

Проєкт побудовано за модульною сервісно-орієнтованою архітектурою:

```
src/
├── services/
│   ├── db.js             # Модуль Persistence / Бази даних (CRUD для авто, користувачів, замовлень)
│   ├── authService.js    # Модуль авторизації, сесій та управління акаунтами
│   ├── bookingService.js # Бізнес-логіка бронювання та розрахунку вартості оренди
│   └── adminService.js   # Адміністративний модуль контролю автопарком та статистикою
├── data/
│   └── cars.js           # Початковий каталог автомобілів
├── main.js               # Контролер інтерфейсу та взаємодії сервісів
└── style.css             # Дизайн-система та стилі компонентів
```

## 🧮 Бізнес-логіка розрахунку оренди (BookingService)

1. **Динамічне визначення тривалості**:
   - Розрахунок кількості діб оренди між датами початку та завершення.
2. **Прогресивна система знижок**:
   - **3–6 діб**: знижка **5%**
   - **7–14 діб**: знижка **10%**
   - **15+ діб**: знижка **15%**
3. **Додаткові сервісні опції**:
   - 🛡️ Повне КАСКО (+$15 / доба)
   - 👶 Дитяче крісло Isofix (+$5 / доба)
   - 🤵 Особистий VIP водій (+$60 / доба)
   - 🌐 5G Wi-Fi роутер (+$4 / доба)
4. **Контроль доступності та статусів авто**:
   - Автоматична зміна статусу авто (`🟢 Вільне` / `🔴 Заброньовано`) при оформленні або скасуванні замовлення.
   - Захист від подвійного бронювання зайнятих автомобілів.
